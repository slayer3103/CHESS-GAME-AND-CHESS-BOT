This Python script is for a **chess game** using the **Pygame library**, handling multiple game screens and interactions. Here's a breakdown of the code:

### **1. Setup and Initialization**
- The script imports required modules for different game screens and utilities.
- It initializes **Pygame**, sets up the game window (`win`), and loads sounds.
- A game loop (`while running`) controls the screen rendering and transitions.

### **2. Screen States**
The game consists of **three states**:
- `WELCOME`: The welcome screen where players can start or quit.
- `CHOOSE_OPPONENT`: A screen where players pick **human** or **computer** as an opponent.
- `GAME`: The actual chess game execution.

### **3. Handling Screen Transitions**
Inside the **main loop**:
- **Welcome Screen**
  - Displays welcome buttons.
  - Handles button clicks: "Start" moves to `CHOOSE_OPPONENT`, "Quit" exits the game.

- **Choose Opponent Screen**
  - Handles player choice between a **human or computer** opponent.
  - Calls `run_game_screen(opponent)`, which runs the chess game.
  - After a game, players can **restart or exit** back to the welcome screen.

- **Game Screen**
  - Runs the game using `run_game(win)`.
  - When the game ends, it exits.

### **4. Event Handling**
- The script captures user **events** (clicks, quits).
- If `pygame.QUIT` is triggered (closing the window), the game exits.
- Click sounds play when interacting with buttons.
  
### **5. Drawing and Updating**
- Each screen's **visuals** are drawn using specific functions.
- `pygame.display.flip()` updates the display on every iteration.

This script **manages transitions** between screens smoothly, ensuring interactions like selecting an opponent and playing a game work correctly. It serves as a framework for an interactive **chess game with a menu system**.
